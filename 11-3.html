現代 Javascript 實務應用

11-3 Trailing Commas

以往早期的 Javascript 版本在陣列和物件屬性最後加上逗號是會出錯。另外 function 放置參數的括號內也會出錯。

Array

版本控制上可以獲得好處。如下範例所示，在看差異時不會連同新增刪減的逗號也一併標記進去，增加可讀性。


 [
    'foo',
+   'baz',
    'bar',
-   'baz',
 ]

結尾逗號不會影響陣列長度


const arr = [
  1,
  2,
  3,
];
arr; // [1, 2, 3]
arr.length; // 3

連續結尾逗號代表新增空白元素。


const arr = [1, 2, 3, , ,];
arr.length; // 5
arr[3]; // undefined

Objects

物件屬性不可連續使用結尾逗號


const object = {
  foo: 'bar',
  baz: 'qwerty',
  age: 42,
};

const object2 = {
  foo: 'bar',
  baz: 'qwerty',
  age: 42,,
};
// SyntaxError: Unexpected token ','

ES6 以後允許 function 使用結尾逗號。以下幾個版本的函式結果相等。


function f(p) {}
function f(p,) {}

(p) => {};
(p,) => {};

其他案例: class 及 物件內 function


class C {
  one(a,) {}
  two(a, b,) {}
}

const obj = {
  one(a,) {},
  two(a, b,) {},
};

Function calls


f(p);
f(p,);

Math.max(10, 20);
Math.max(10, 20,);

Illegal trailing commas


function f(,) {} // SyntaxError: missing formal parameter
(,) => {};       // SyntaxError: expected expression, got ','
f(,)             // SyntaxError: expected expression, got ','

function f(...p,) {} // SyntaxError: parameter after rest parameter
(...p,) => {}        // SyntaxError: expected closing parenthesis, got ','

Trailing commas in destructuring


// array destructuring with trailing comma
[a, b,] = [1, 2];

// object destructuring with trailing comma
const o = {
  p: 42,
  q: true,
};
const { p, q, } = o;

也無法用在展開運算子


const [a, ...b,] = [1, 2, 3];
// SyntaxError: rest element may not have a trailing comma

Trailing commas in JSON

JSON 不支援。


JSON.parse('[1, 2, 3, 4, ]');
JSON.parse('{"foo" : 1, }');
// SyntaxError JSON.parse: unexpected character
// at line 1 column 14 of the JSON data
      

JSON.parse('[1, 2, 3, 4 ]');
JSON.parse('{"foo" : 1 }');

Trailing commas in named imports and named exports

Named imports


import {
  A,
  B,
  C,
} from 'D';

import { X, Y, Z, } from 'W';

import { A as B, C as D, E as F, } from 'Z';

Named exports


export {
  A,
  B,
  C,
};

export { A, B, C, };

export { A as B, C as D, E as F, };

Quantifier prefix


/x{2}/; // Exactly 2 occurrences of 'x'; equivalent to /xx/
/x{2,}/; // At least 2 occurrences of 'x'; equivalent to /xx+/
/x{2,4}/; // 2 to 4 occurrences of 'x'; equivalent to /xxx?x?/
