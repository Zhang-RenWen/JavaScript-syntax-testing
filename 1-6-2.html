再回到根本問題：為什麼不能直接在 JSX 中使用 if.eise 實現條件繪製，而只能借用函數運輯實現呢？實際上，我們都知道 JSX 會被編譯為 React. createElenent。直白地說，React createElenent 的底層邏輯是無法執行 JavaScript 程式的，只能繪製一個結果。因此，JSX 中除了 JavaScript 運算式，不能直接寫 JavaScript 語法。準確來講，JSX 只是函數呼叫和運算式的語法糖。

雖然 JSX 只是函數呼叫和運算式的語法糖，但是JSX 仍然具有強大的功能且能夠使用靈活。React 元件重複使用最流行的方式都是以 JSX 能力為基礎的， 舉例來説，HOC（Higher Order Component，高階元件）和 render prop 模式， render prop 模式的實現程式如下
