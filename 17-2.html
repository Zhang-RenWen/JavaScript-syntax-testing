17-2 Array.prototype.toReversed

Array.prototype.toReversed 是由 Array 實例中的方法 Array.prototype.reverse 複製而來的新方法，一樣是給陣列做排序。最大差異在於 toReversed 會創建新陣列，不會去改動原本的資料。

比較 reverse 和 toReversed

reverse


const items = [1, 2, 3];
console.log(items); // [1, 2, 3]

const reversedItems = items.reverse();
console.log(reversedItems); // [3, 2, 1]
console.log(items); // [3, 2, 1]

toReversed


const items = [1, 2, 3];
console.log(items); // [1, 2, 3]

const reversedItems = items.toReversed();
console.log(reversedItems); // [3, 2, 1]
console.log(items); // [1, 2, 3]

Using toReversed() on sparse arrays 處理有空值的陣列


console.log([1, , 3].toReversed()); // [3, undefined, 1]
console.log([1, , 3, 4].toReversed()); // [4, 3, undefined, 1]

calling toReversed 處理非陣列物件

下列範例提供累陣列物件給 toReversed() 方法。可以發現 toReversed 會讀取 this 中的 length 屬性。 toReversed 會巡訪每一個有整數 key 的屬性，並且取出屬性值按照 key 值範圍 length - 1 and 0 由大到小依序放入要回傳的新陣列中。 超出 length 的部分會用 undefined 填上。 


const arrayLike = {
  length: 3,
  unrelated: "foo",
  2: 4,
};
console.log(Array.prototype.toReversed.call(arrayLike));
// [4, undefined, undefined]
// The '0' and '1' indices are not present so they become undefined
        

const arrayLike = {
  length: 1,
  2: 4,
};
console.log(Array.prototype.toReversed.call(arrayLike));
// [undefined]
        

const arrayLike = {
  length: 2,
  2: 4,
};
console.log(Array.prototype.toReversed.call(arrayLike));
// [undefined, undefined]
        

const arrayLike = {
  length: 3,
  2: 4,
};
console.log(Array.prototype.toReversed.call(arrayLike));
// [4, undefined, undefined]
        

const arrayLike = {
  length: 4,
  2: 4,
};
console.log(Array.prototype.toReversed.call(arrayLike));
// [undefined, 4, undefined, undefined]
        

const arrayLike = {
  length: 4,
  1: 5,  
  2: 4,
  0: 3 
};
console.log(Array.prototype.toReversed.call(arrayLike));
//  [undefined, 4, 5, 3]
