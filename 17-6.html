17-6 Array.prototype.toSpliced

toSpliced() 實例的方法是Array該方法的複製 splice() 版本。它回傳一個新陣列，其中一些元素在給定索引處被刪除和/或替換。

語法


toSpliced(start)
toSpliced(start, deleteCount)
toSpliced(start, deleteCount, item1)
toSpliced(start, deleteCount, item1, item2)
toSpliced(start, deleteCount, item1, item2, /* …, */ itemN)

參數

start

開始更改陣列的從零開始的索引， 轉換為整數 。

  • 負索引從陣列末尾開始倒數 — 如果 使用 , start < 0 。 start + array.length
  • 如果 省略 start < -array.length 或，則使用。 start 0
  • 如果 start >= array.length ，則不會刪除任何元素，但該方法將充當添加函數，添加提供的盡可能多的元素。
deleteCount Optional

一個整數，指示陣列中要從中刪除的元質數 start 。

如果 deleteCount 省略 ，或者其值大於或等於指定位置之後的元素個數，則 刪除 start 從陣列末尾開始的所有元素。 start 但是，如果您希望傳遞任何 itemN 參數，則應該傳遞 Infinity as deleteCount 以刪除 後的所有元素 start ，因為顯式 undefined 會 轉換 為 0 。

如果 deleteCount 是 0 或負數，則不刪除任何元素。 在這種情況下，您應該至少指定一個新元素（見下文）。

item1 , …, itemN 可選

要添加到陣列的元素，從 開始 start 。

如果不指定任何元素， toSpliced() 只會從陣列中刪除元素。

回傳值

start 一個新陣列，由、 item1 、 item2 、 …、之前的所有元素 itemN 以及 之後的所有元素 組成 start + deleteCount 。

範例

刪除、添加和替換元素

您可以使用 toSpliced() 來刪除、添加和替換陣列中的元素，並比使用 slice() 和更有效地創建新陣列 concat() 。 


const months = ["Jan", "Mar", "Apr", "May"];

// Inserting an element at index 1
const months2 = months.toSpliced(1, 0, "Feb");
console.log(months2); // ["Jan", "Feb", "Mar", "Apr", "May"]

// Deleting two elements starting from index 2
const months3 = months2.toSpliced(2, 2);
console.log(months3); // ["Jan", "Feb", "May"]

// Replacing one element at index 1 with two new elements
const months4 = months3.toSpliced(1, 1, "Feb", "Mar");
console.log(months4); // ["Jan", "Feb", "Mar", "May"]

// Original array is not modified
console.log(months); // ["Jan", "Mar", "Apr", "May"]

在稀疏陣列上使用 toSpliced()

該 toSpliced() 方法始終創建一個密集陣列。 


const arr = [1, , 3, 4, , 6];
console.log(arr.toSpliced(1, 2)); // [1, 4, undefined, 6]

對類陣列物件調用 toSpliced()

該 toSpliced() 方法讀取 length 的屬性 this 。 然後，它讀取所需的整數鍵屬性並將它們寫入新陣列中。 


const arrayLike = {
  length: 3,
  unrelated: "foo",
  0: 5,
  2: 4,
};
console.log(Array.prototype.toSpliced.call(arrayLike, 0, 1, 2, 3));
// [2, 3, undefined, 4]
